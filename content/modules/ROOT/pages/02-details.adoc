== Using the Web Terminal

All CLI commands in this workshop are run in the OpenShift Web Terminal. The Web Terminal runs directly in your browser â€” no local `oc` CLI installation is required.

. Open the OpenShift console by navigating to pass:[<a href="%OPENSHIFT_CONSOLE_URL%" target="_blank">%OPENSHIFT_CONSOLE_URL%</a>].

. When prompted, log in with your username `{OPENSHIFT_USERNAME}` and password `{OPENSHIFT_PASSWORD}`.

. Click the command-line terminal icon (*>_*) in the upper-right corner of the console masthead.
+
image::web-terminal-icon.png[Web Terminal icon in the OpenShift console masthead]

. Select the project `terminal-{OPENSHIFT_USERNAME}` from the *Project* dropdown if prompted.

. Click *Start* to launch the terminal session.

. The Web Terminal comes preinstalled with `oc`, `kubectl`, and other CLI tools, and is automatically logged in with your user credentials. No `oc login` is required.
+
image::web-terminal-panel.png[Web Terminal open in the OpenShift console]
. When started the terminal looks like the following
+
image::web-terminal-panel2.png[Web Terminal open in the OpenShift console]

[TIP]
====
If the Web Terminal session times out or disconnects, click the terminal icon again to start a new session. Your credentials will be automatically restored.
====

=== Verify your environment

Vrify your terminal by running these commands:

[source,bash,role=execute]
----
oc whoami
----

Expected output: Your username (`{OPENSHIFT_USERNAME}`)

[source,bash,role=execute]
----
oc version
----

Expected output: OpenShift client and server version information
