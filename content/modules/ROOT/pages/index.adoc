= Network Policies Introduction

Welcome to the Network Policies Introduction workshop!

== What you'll learn

In this workshop, you will:

* Understand how pod networking works in Kubernetes and OpenShift
* Create ingress and egress network policies
* Test and validate network policy enforcement
* Troubleshoot connectivity issues

== Who this is for

This workshop is designed for developers and platform engineers who want to understand and implement network security in OpenShift.

**Experience level**: Beginner

== Prerequisites

Before starting this workshop, you should have:

* Basic familiarity with Kubernetes concepts (pods, namespaces, and services)
* Command line experience (Linux/Unix)
* Access to a web browser

== Workshop environment

You will work in a pre-configured OpenShift cluster with all necessary tools installed.

== Estimated time

This workshop will take approximately **1 hour** to complete.

* **Module 1**: Understanding network policy basics (20 minutes)
* **Module 2**: Implementing ingress and egress policies (25 minutes)
* **Module 3**: Cross-namespace policies and troubleshooting (15 minutes)
